# What is Snowflake Architecture?

Snowflake is built using 3 layers:


Cloud Services Layer
        ↑
Query Processing Layer
        ↑
Storage Layer


1. Storage Layer (Stores Data)

This is where your actual data is saved.

Example:

Students      Table
ID	      Name
1	      Roohi
2	      Rahul

This data is stored in the cloud.

Since your Snowflake account uses AWS:

Snowflake
    ↓
AWS S3

Snowflake stores data behind the scenes in AWS S3 buckets.

You don't manage S3 yourself.

Snowflake handles everything automatically.


# Traditional Database vs Snowflake;

Traditional Database

Stores data row by row:

1 | Roohi | Snowflake
2 | Rahul | Python
3 | Sara  | SQL

Snowflake-Stores data column by column:

ID
1
2
3

NAME
Roohi
Rahul
Sara

COURSE
Snowflake
Python
SQL

This is called:

Columnar Storage

Benefits:

✅ Faster analytics

✅ Better compression

✅ Lower storage cost


2. Query Processing Layer (The Muscle 💪)

This layer executes SQL queries.

Example:

SELECT * FROM students;

Someone has to process this query.

What is a Warehouse?

A Warehouse is a virtual computer.

Think:

SQL Query
     ↓
 Warehouse
     ↓
 Result

In your account:

COMPUTE_WH

is your warehouse.


Why Do We Need Warehouses?

Example:

SELECT * FROM students;

Snowflake must:

Read data
Process query
Return result

This requires CPU and memory.

Warehouse provides that computing power.


3. Cloud Services Layer (The Brain 🧠)

This layer manages everything.

Responsibilities:

Security
User accounts
Roles
Query optimization
Metadata
Permissions

Example:

When you run:

CREATE TABLE students (
    id INTEGER
);

Cloud Services checks:

Does ACCOUNTADMIN have permission?

If yes:

Query executes

If no:

Permission denied

❌ Not for making one query faster



# Standard Scaling Policy

Default policy.

SCALING_POLICY = 'STANDARD'

Behavior:

New Query Arrives
↓
Need More Compute?
↓
Immediately Start New Cluster

Priority:

Performance First
Cost Second

Example:

9:00 AM → Heavy workload
9:01 AM → New cluster started

Users experience less waiting.



# Economy Scaling Policy

SCALING_POLICY = 'ECONOMY'

Behavior:

New Queries Arrive
↓
Snowflake Waits
↓
Checks if load continues
↓
Then starts new cluster

Priority:

Cost First
Performance Second


Example

Heavy workload starts:

9:00 AM

Standard Policy:

9:01 AM
New Cluster Started

Economy Policy:

9:01 AM
Wait

9:02 AM
Wait

9:03 AM
Wait

If workload continues,
start cluster

So fewer credits are consumed.


Standard vs Economy

Feature	                           Standard	       Economy

Starts new cluster quickly	     ✅	                 ❌
Best performance	             ✅	                 ❌
Saves credits	                     ❌      	         ✅
Default policy	                     ✅	                 ❌
Good for dashboards	             ✅	             Sometimes

# What is a Data Warehouse?

A Data Warehouse is a central place where data from different sources is collected, cleaned, organized, and stored for reporting, analytics, dashboards, and business decisions.

Example

Suppose a company has:

HR Database → Employee data
Sales Database → Sales transactions
Finance Database → Expenses and revenue
CSV Files → Customer feedback

Instead of analyzing each source separately, all data is brought into a Data Warehouse (Snowflake).

ETL Process:-

1. Extract (E)

Take data from source systems.

HR Database  → Employee Data
Sales DB     → Sales Data
CSV Files    → Customer Data


2. Transform (T)

Clean and prepare data.

Examples:

Rahul   → RAHUL
50000.0 → 50000
Remove duplicate records
Fix missing values


3. Load (L)

Load the cleaned data into Snowflake tables.

EMPLOYEES
SALES
CUSTOMERS

Data Warehouse Layers

Layer 1: Raw Layer (Staging Area)

Data is copied exactly as received.

RAW_EMPLOYEES
RAW_SALES
RAW_CUSTOMERS

No cleaning yet.

Layer 2: Integration Layer

Data is cleaned and transformed.

RAW_EMPLOYEES
      ↓
Cleaned Employees

RAW_SALES
      ↓
Cleaned Sales

Relationships are created here.

Layer 3: Access Layer

Final business-ready tables.

EMPLOYEE_REPORT
SALES_REPORT
CUSTOMER_REPORT



Tools like Power BI connect here.

Source Systems
      ↓
ETL Process
      ↓
Snowflake
      ↓
Power BI / Tableau / Reports

# Snowflake Pricing-



Snowflake charges mainly for 2 things:


Snowflake Cost
   │
   ├── Compute Cost 💻
   └── Storage Cost 💾



1. Compute Cost (Major Cost)

Compute means the processing power used to run queries.

Example:

SELECT * FROM EMPLOYEES;

To execute this query, Snowflake uses a Warehouse.

You pay only when the warehouse is running.


Compute Cost Depends On


A. Warehouse Size

Warehouse Size	   Credits / Hour
X-Small          	1
Small	                2
Medium	                4
Large	                8
X-Large	               16

Notice:

X-Small = 1 credit
Small   = 2 credits
Medium  = 4 credits
Large   = 8 credits

Every size doubles the compute power and doubles the credits.



B. Running Time

Suppose:

Warehouse = Small
Credits/hr = 2

If it runs for:

30 minutes

Then:

2 × (30/60)

= 1 credit

So Snowflake bills per second (after a minimum of 1 minute).



Example

Warehouse:

SMALL

Runs for:

15 minutes

Calculation:

2 credits × (15/60)

= 0.5 credits



Auto Suspend Saves Money

Example:

ALTER WAREHOUSE MY_WH
SET AUTO_SUSPEND = 60;

Meaning:

No query for 60 seconds
       ↓
Warehouse automatically stops
       ↓
No compute charges


This is why Auto Suspend is important.



2. Storage Cost

This is the cost of storing data.

Example:

employees.csv
sales.csv
customer_data.csv

stored inside Snowflake.

You pay based on:

Total GB/TB stored

Example:

100 GB data

Storage cost is usually much lower than compute cost.



Cloud Services Cost

Snowflake also performs:

Metadata management
Query optimization
Authentication
Access control

These belong to the Cloud Services Layer.

Normally:

Cloud Service Cost
≤ 10% of warehouse usage

Snowflake usually does not charge separately.


Serverless Compute

Used by features like:

Snowpipe
Search Optimization
Automatic Clustering

Example:

New CSV file arrives
      ↓
Snowpipe automatically loads it
      ↓
Snowflake provides compute automatically


No warehouse management needed.


#  Snowflake Pricing-


Snowflake charges for 3 things:

Compute Cost (Warehouses)
Storage Cost (Data stored)
Data Transfer Cost (Moving data out)

1. Compute Cost

This is the main cost.

Whenever you run queries, Snowflake uses a warehouse.

Example:

SELECT * FROM EMPLOYEES;

If your warehouse is running:

XS Warehouse
1 hour running= 1 credit

If the warehouse runs for only 30 minutes= 0.5 credit

Larger warehouses consume more credits.

Warehouse	Credits/Hour
XS	             1
S	             2
M	             4
L	             8


2. Storage Cost

Snowflake stores your data in cloud storage (AWS/Azure/GCP).

Example:

EMPLOYEES table
CUSTOMERS table
SALES table

All these tables occupy storage.

Suppose:

Total Data = 1 TB

Storage cost might be:

$40 per TB per month

(Actual price depends on region and cloud provider.)


Important

Snowflake compresses data.

Example:

Original CSV = 100 GB
After Compression = 40 GB

You pay only for:

40 GB not 100 GB


3. Data Transfer Cost

Data In (Ingress)

Uploading data into Snowflake:

Laptop → Snowflake
AWS S3 → Snowflake
CSV → Snowflake

✅ FREE


Data Out (Egress)

Moving data out of Snowflake:

Snowflake → Another Cloud
Snowflake → Another Region
Snowflake → Another Snowflake Account

❌ Charged

No charge
